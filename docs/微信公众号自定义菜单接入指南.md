# 微信公众号自定义菜单接入指南

## 核心原理大白话解释

微信官方要求：**要让公众号底部出现菜单，你必须把写好的“菜单图纸（JSON代码）”，通过一个专门的“邮局接口（`/cgi-bin/menu/create`）”寄给微信的中央服务器。** 微信服务器审核图纸没问题后，就会下发给所有关注你公众号的手机。

既然我们项目中已经集成了 `weixin-java-mp` (WxJava SDK)，这个 SDK 已经帮我们把“找邮局、盖公章（鉴权）”的脏活累活干完了。我们只需要**写一段 Java 测试代码**，把你的那段 JSON 字符串交给 SDK，它就会自动帮你送到微信服务器。

## 具体操作步骤

你提供的 JSON 菜单结构包含 3 个主菜单：“作文批改”、“自助充值”、“我的”，并且包含带有网页鉴权链接（`oauth2/authorize`）的跳转（`view`）以及点击事件（`click`）。

我已经为你编写了一个**一键发布菜单的测试类**。你只需要按照以下 3 步操作：

### 第一步：确认你的测试类代码已就位
我已经自动在你的项目中创建了测试类：`src/test/java/com/dong/easyeval/wx/WxMenuPublisherTest.java`。

这个类会读取你提供的 JSON 格式代码，并利用 `WxMpService` 将其推送到微信服务器。

### 第二步：运行测试类
1. 打开 IDEA，找到 `src/test/java/com/dong/easyeval/wx/WxMenuPublisherTest.java` 文件。
2. 找到 `publishMenuFromJson()` 方法。
3. 点击方法左侧的 **绿色运行箭头 (Run 'publishMenuFromJson()')**。
4. 观察控制台的输出。如果看到类似以下的绿色字样：
   ```text
   🎉 恭喜！菜单创建成功。
   微信服务器返回结果：null （注：WxJava 成功时通常返回 null 或空字符串，只要不抛异常就是成功）
   ```
   这就说明你的菜单图纸已经成功送达微信中央服务器了！

### 第三步：在手机上查看效果
⚠️ **非常重要：微信客户端对菜单有缓存机制！**
就算你在后台创建成功了，手机上可能也不会立刻变样。
*   **最快生效办法**：在手机上取消关注你的测试公众号，然后再重新扫码关注。点进去，底部菜单就出来了！
*   **慢办法**：什么都不做，等最多 24 小时，微信自己会刷新。

## 常见疑问解答 (Q&A)

### 1. 运行一次是永久的吗？还是我服务器关了就没了？
**答案：是永久的！就算你的服务器关了、甚至电脑砸了，菜单也还在。**
**原理大白话**：你运行代码，就好比你写了一封信（菜单配置）寄到了“微信总局（微信服务器）”。微信总局把你的菜单记录在他们自家的数据库里了。你的服务器关机，只是说明你“不寄信了”，但微信那边早就把菜单存好了，只要用户打开公众号，微信总局就会把菜单发给用户看。

### 2. 那我要更改菜单又该怎么办？
**答案：改完 JSON 代码，再重新运行一次刚才的测试类就行了！**
**原理大白话**：微信的规则是“覆盖式更新”。如果你想改菜单（比如把“自助充值”改成“马上充值”，或者换个链接），你只需要在 `WxMenuPublisherTest.java` 里面把那个 `menuJson` 里的字改掉，然后**再点击一次绿色的运行按钮**。
当你再次调用接口发送时，微信总局一看：“哦，这个公众号又寄来新菜单了”，它就会立刻把旧菜单删掉，替换成你这次发过去的新菜单。
*(别忘了：每次改完重新运行后，手机上也要**重新关注**才能立刻看到新变化！)*

## 菜单背后的交互逻辑（下一步要做什么？）

当菜单在手机上显示出来后，用户点击菜单时，微信会跟我们的服务器产生互动。根据你 JSON 里的配置，分为两种情况：

### 1. 网页跳转类型 (`type: "view"`)
*   **比如**：“作文批改”、“我的卡券”、“批改记录”。
*   **发生了什么**：用户点击后，微信直接在内置浏览器里打开你配置的 URL（比如 `https://open.weixin.qq.com/connect/oauth2/...`）。
*   **我们要做什么**：你配置的是微信的 OAuth2.0 网页授权链接，这意味着微信会先跳到一个授权中转页，拿到一个 `code`，然后再重定向（`redirect_uri`）到你的 `11ir9803988vm.vicp.fun` 服务器。**我们需要在后续的开发中，写 Controller 来接收这个 `code`，并用它换取用户的 OpenID。**

### 2. 点击推送事件类型 (`type: "click"`)
*   **比如**：“充值方法” (`key: "REDEEM_VOUCHER"`)、“唯一标识” (`key: "MY_OPENID"`)、“剩余额度” (`key: "REMAINING_AMOUNT"`)。
*   **发生了什么**：用户点击后，微信服务器会立刻给我们的 `WxResponse` (之前写的用于验证的 Controller) 发送一个 **POST 请求**。请求里面是一个 XML 格式的信件，告诉我们：“有个用户点击了 `REDEEM_VOUCHER` 这个按钮”。（这就是《微信开发文档》第 3.1 节描述的内容）
*   **我们要做什么**：我们需要修改 `WxResponse.java`，增加对 `POST` 请求的处理能力。我们要拆开微信发来的 XML 信件，看看里面的 `EventKey` 是什么。如果是 `MY_OPENID`，我们就查出他的 OpenID 并打包成一段 XML 文本回复过去，微信就会在对话框里发给用户。

> **总结：** 你现在的首要任务是运行我刚写好的 `WxMenuPublisherTest`。菜单出来后，我们再来处理 `type="click"` 按钮的后台回复逻辑！
